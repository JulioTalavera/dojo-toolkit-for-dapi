dijit.tree.TreeStoreModel (version 1.7)

Object » dijit.tree.TreeStoreModel

Usage:

var foo = new dijit.tree.TreeStoreModel(/* Object */ args);

Implements dijit.Tree.model connecting to a dojo.data store with a single root item. Any methods passed into the constructor will override the ones defined here.

Examples:

Example 1

	{id:'ROOT'}

Method Summary

  • constructor(args) returns dijit.tree.TreeStoreModel Passed the arguments listed above (store, etc)
  • destroy()
  • fetchItemByIdentity(keywordArgs)
  • getChildren(parentItem, onComplete, onError) Calls onComplete() with array of child items of given parent item, all loaded.
  • getIdentity(item) returns Object
  • getLabel(item) returns String Get the label for an item
  • getRoot(onItem, onError) Calls onItem with the root item for the tree, possibly a fabricated item. Calls onError on error.
  • isItem(something) returns Boolean
  • mayHaveChildren(item) Tells if an item has or may have children. Implementing logic here avoids showing +/- expando icon for nodes that we know don't have children. (For efficiency reasons we may not want to check if an element actually has children until user clicks the expando node)
  • newItem(args, parent, insertIndex) Creates a new item. See `dojo.data.api.Write` for details on args. Used in drag & drop when item from external source dropped onto tree.
  • pasteItem(childItem, oldParentItem, newParentItem, bCopy, insertIndex) Move or copy an item from one parent item to another. Used in drag & drop

Event Summary

  • onChange(item) Callback whenever an item has changed, so that Tree can update the label, icon, etc. Note that changes to an item's children or parent(s) will trigger an onChildrenChange() so you can ignore those changes here.
  • onChildrenChange(parent, newChildrenList) Callback to do notifications about new, updated, or deleted items.
  • onDelete(item) Callback when an item has been deleted.
  • onDeleteItem(item) Handler for delete notifications from underlying store
  • onNewItem(item, parentInfo) Handler for when new items appear in the store, either from a drop operation or some other way. Updates the tree view (if necessary).
  • onSetItem(item, attribute, oldValue, newValue) Updates the tree view according to changes in the data store.
