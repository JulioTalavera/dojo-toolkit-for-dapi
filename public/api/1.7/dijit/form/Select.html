dijit.form.Select (version 1.7)

Object » dijit._FormSelectWidget » dijit.form.Select

Usage:

var foo = new dijit.form.Select();

This is a "styleable" select box - it is basically a DropDownButton which can take a <select> as its input.

dijit._HasDropDown

Property Summary

  • dijit.form.Select.containerNode
  • dijit.form.Select.srcNodeRef
  • dropDown The widget to display as a popup. This widget *must* be defined before the startup function is called.
  • dropDownPosition This variable controls the position of the drop down. It's an array of strings with the following values: * before: places drop down to the left of the target node/widget, or to the right in the case of RTL scripts like Hebrew and Arabic * after: places drop down to the right of the target node/widget, or to the left in the case of RTL scripts like Hebrew and Arabic * above: drop down goes above target node * below: drop down goes below target node The list is positions is tried, in order, until a position is found where the drop down fits within the viewport.

Method Summary

  • _addOptionItem(option) For the given option, add an option to our dropdown. If the option doesn't have a value, then a separator is added in that place.
  • _fillContent() Set the value to be the first, or the selected index
  • _getChildren()
  • _getMenuItemForOption(option) For the given option, return the menu item that should be used to display it. This can be overridden as needed
  • _loadChildren(loadMenuItems) Resets the menu and the length attribute of the button - and ensures that the label is appropriately set.
  • _setDisabledAttr(value)
  • _setDisplay(newDisplay) sets the display for the given value (or values)
  • _setRequiredAttr(value)
  • _setStyleAttr(value)
  • _setValueAttr(value)
  • buildRendering()
  • closeDropDown(focus) Closes the drop down on this widget
  • constructor() returns dijit.form.Select A reference to the constructor function for this object.
  • destroy()
  • isLoaded() Returns true if the dropdown exists and it's data is loaded. This can be overridden in order to force a call to loadDropDown().
  • isValid(isFocused) returns handle value is null or undefined Whether or not this is a valid value. The only way a Select can be invalid is when it's required but nothing is selected.
  • loadAndOpenDropDown() Creates the drop down if it doesn't exist, loads the data if there's an href and it hasn't been loaded yet, and then opens the drop down. This is basically a callback when the user presses the down arrow button to open the drop down.
  • loadDropDown(loadCallback) populates the menu
  • openDropDown() Opens the dropdown for this widget. To be called only when this.dropDown has been created and is ready to display (ie, it's data is loaded).
  • postCreate() stop mousemove from selecting text on IE to be consistent with other browsers
  • postMixInProperties() set the missing message
  • reset() Overridden so that the state will be cleared.
  • toggleDropDown() Callback when the user presses the down arrow button or presses the down arrow key to open/close the drop down. Toggle the drop-down widget; if it is up, close it, if not, open it
  • uninitialize(preserveDom)
  • validate(isFocused) Called by oninit, onblur, and onkeypress, and whenever required/disabled state changes

Event Summary

  • _onBlur() Called magically when focus has shifted away from this widget and it's dropdown
  • _onDropDownClick(e)
  • _onDropDownMouseDown(e) Callback when the user mousedown's on the arrow icon
  • _onDropDownMouseUp(e) Callback when the user lifts their mouse after mouse down on the arrow icon. If the drop down is a simple menu and the mouse is over the menu, we execute it, otherwise, we focus our drop down widget. If the event is missing, then we are not a mouseup event. This is useful for the common mouse movement pattern with native browser <select> nodes: 1. mouse down on the select node (probably on the arrow) 2. move mouse to a menu item while holding down the mouse button 3. mouse up. this selects the menu item as though the user had clicked it.
  • _onFocus()
  • _onKey(e) Callback when the user presses a key while focused on the button node
  • _onKeyUp()
