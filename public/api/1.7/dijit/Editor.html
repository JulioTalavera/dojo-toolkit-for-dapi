dijit.Editor (version 1.7)

Object » RichText » dijit.Editor
dojo.require("dijit.Editor");
Defined in dijit/Editor.js

Usage:

var foo = new dijit.Editor();

This widget provides basic WYSIWYG editing features, based on the browser's underlying rich text editing capability, accompanied by a toolbar (dijit.Toolbar). A plugin model is available to extend the editor's capabilities as well as the the options available in the toolbar. Content generation may vary across browsers, and clipboard operations may have different results, to name a few limitations. Note: this widget should not be used with the HTML <TEXTAREA> tag -- see dijit._editor.RichText for details.

  • _pasteImpl() Over-ride of paste command control to make execCommand cleaner
  • _restoreSelection() Re-select the text specified in _savedSelection attribute; see _saveSelection().
  • _saveSelection() Save the currently selected text in _savedSelection attribute
  • _setDisabledAttr(value)
  • _setStateClass()
  • addPlugin(plugin, index) takes a plugin name as a string or a plugin instance and adds it to the toolbar and associates it with this editor instance. The resulting plugin is added to the Editor's plugins array. If index is passed, it's placed in the plugins array at that index. No big magic, but a nice helper for passing in plugin names via markup.
  • beginEditing(cmd) Called to note that the user has started typing alphanumeric characters, if it's not already noted. Deals with saving undo; see editActionInterval parameter.
  • constructor() returns dijit.Editor Runs on widget initialization to setup arrays etc.
  • destroy()
  • endEditing(ignore_caret) Called to note that the user has stopped typing alphanumeric characters, if it's not already noted. Deals with saving undo; see editActionInterval parameter.
  • execCommand(cmd) Main handler for executing any commands to the editor, like paste, bold, etc. Called by plugins, but not meant to be called by end users.
  • layout() Called from `dijit.layout._LayoutWidget.resize`. This shouldn't be called directly
  • postCreate()
  • postMixInProperties() Extension to make sure a deferred is in place before certain functions execute, like making sure all the plugins are properly inserted.
  • queryCommandEnabled(cmd) Returns true if specified editor command is enabled. Used by the plugins to know when to highlight/not highlight buttons.
  • redo() Handler for editor redo (ex: ctrl-y) operation
  • replaceValue(html) over-ride of replaceValue to support custom undo and stack maintenance.
  • resize(size) Resize the editor to the specified size, see `dijit.layout._LayoutWidget.resize`
  • undo() Handler for editor undo (ex: ctrl-z) operation

  • onBeforeDeactivate(e) Called on IE right before focus is lost. Saves the selected range.

  • onKeyDown(e) Handler for onkeydown event.
