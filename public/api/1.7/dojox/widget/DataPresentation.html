DataPresentation

A widget that connects to a data store in a simple manner, and also provides some additional convenience mechanisms for connecting to common data sources without needing to explicitly construct a Dojo data store. The widget can then present the data in several forms: as a graphical chart, as a tabular grid, or as display panels presenting meta-data (title, creation information, etc) from the data. The widget can also create and manage several of these forms in one simple construction.

Note: this is a first experimental draft and any/all details are subject to substantial change in later drafts.

  • cancelRefresh() Cancels any and all outstanding data refreshes
  • constructor(node, args) returns dojox.widget.DataPresentation Set up properties and initialize. arguments:
  • destroy() Destroys the widget and all components and resources.
  • getChartWidget() Returns the chart widget (if any) created if the type is "chart" or the "chartNode" property was supplied.
  • getGridWidget() Returns the grid widget (if any) created if the type is "grid" or the "gridNode" property was supplied.
  • refresh() If a URL or data has been supplied, refreshes the presented data from the URL or data. If a refresh interval is also set, the periodic refresh is restarted. If a URL or data was not supplied, this method has no effect.
  • renderChartWidget() Renders the chart widget (if any). This method is called whenever a chart widget is created or configured, and may be connected to.
  • renderGridWidget() Renders the grid widget (if any). This method is called whenever a grid widget is created or configured, and may be connected to.
  • setData(data, refreshInterval) Sets the data to be presented, and an optional refresh interval in milliseconds (0=no refresh)
  • setPreparedStore(store, query, queryOptions) Sets the store and query.
  • setStore(store, query, queryOptions)
  • setURL(url, urlContent, refreshInterval) Sets the URL to fetch data from, with optional content supplied with the request, and an optional refresh interval in milliseconds (0=no refresh)
