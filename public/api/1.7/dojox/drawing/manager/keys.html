dojox.drawing.manager.keys (version 1.7)

Object » dojox.drawing.manager.keys
dojo.require("dojox.drawing.manager.keys");
Defined in dojox/drawing/manager/keys.js

A singleton, master object that detects keyboard keys and events Connect to it like: dojo.connect(this.keys, "onEnter", ....);

Method Summary

  • dojox.drawing.manager.keys._getLetter(evt)
  • dojox.drawing.manager.keys._mixin(evt) Internal. Mixes in key events.
  • dojox.drawing.manager.keys.editMode(_isedit) Relinquishes control of events to another portion of Drawing; namely the TextBlock.
  • dojox.drawing.manager.keys.enable(_enabled) Enables or disables key events, to relinquish control to something outside of Drawing; input fields for example. You may need to call this directly if you are using textareas or contenteditables. NOTE: See scanForFields
  • dojox.drawing.manager.keys.init() Initialize the keys object a little extra time is needed in some browsers
  • dojox.drawing.manager.keys.register(options) Register an object and callback to be notified of events. NOTE: Not really used in code, but should work. See manager.mouse for similar usage
  • dojox.drawing.manager.keys.scanForFields() Scans the document for inputs and calls this automatically. However you may need to call this if you create inputs after the fact.

Event Summary

  • dojox.drawing.manager.keys.onArrow(evt) Event fires when an Arrow key is released You will have to further check if evt.keyCode is 37,38,39, or 40
  • dojox.drawing.manager.keys.onDelete(evt) Event fires when Delete key is released
  • dojox.drawing.manager.keys.onEnter(evt) Event fires when Enter key is released
  • dojox.drawing.manager.keys.onEsc(evt) Event fires when ESC key is released
  • dojox.drawing.manager.keys.onKeyDown(evt) Event fires when any key is pressed
  • dojox.drawing.manager.keys.onKeyUp(evt) Event fires when any key is released
