dojo.require("dojox.grid.enhanced.plugins.filter.FilterLayer");

Usage:

var foo = new dojox.grid.enhanced.plugins.filter.ClientSideFilterLayer(/* */ args);

Add a client side filter layer on top of the data store, so any filter expression can be applied to the store.

Property Summary

  • _items Cached items (may contain holes)

Method Summary

  • _addCachedItems(items, filterStartIdx) Add data items to the cache. The insert point is at *filterStartIdx*
  • _applyFilter(datarow, rowIndex) Apply the filter to a row of data
  • _completeQuery(userRequest) Logically, the user's query is completed here, i.e., all the filtered results are ready. (or their index mappings are ready)
  • _defaultGetter(datarow, colName, rowIndex, store)
  • _doFilter(items, startIdx, userRequest) Use the filter expression to filter items. Survived items are stored in this._items. The given items start from "startIdx" in the original store.
  • _fetch(userRequest, filterRequest) Implement _StoreLayer._fetch
  • _hasReachedStoreEnd() returns Boolean Check whether all the items in the original store have been fetched.
  • constructor(args) returns dojox.grid.enhanced.plugins.filter.ClientSideFilterLayer A reference to the constructor function for this object.
  • fetchAllOnFirstFilter(toFetchAll) returns Boolean The get/set function for fetchAll.
  • filterDef(filter) returns _ConditionExpr See _FilterLayerMixin.filterDef
  • invalidate() Clear all the status information of this layer
  • setGetter(getter) Set the user defined way to retrieve data from store.

Event Summary
