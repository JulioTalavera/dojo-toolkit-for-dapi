dojox.timing (version 1.7)

Object » dojox.timing

A set of objects to perform advanced time-based tasks, including a basic Timer.

Property Summary

  • dojox.timing.threadPriorities
  • dojox.timing.threadStates

Method Summary

  • dojox.timing.Sequence() This class provides functionality to really sequentialize function calls. You need to provide a list of functions and some parameters for each (like: pauseBefore) and they will be run one after another. This can be very useful for slideshows or alike things.
  • dojox.timing.Streamer(input, output, interval, minimum, initialData) the function executed when the internal queue reaches minimumSize
  • dojox.timing.Thread(fn, priority)
  • dojox.timing.Timer(interval) Timer object executes an "onTick()" method repeatedly at a specified interval. repeatedly at a given interval.
  • dojox.timing.doLater(conditional, context, interval) returns Boolean Check if a parameter is ready, and if not, "do later". doLater will ping the parameter until it evaluates to something (truthy). It thens calls the caller with original arguments, using the supplied context or window.

dojox.timing.doLater
Defined by dojox.timing
Returns Boolean

dojox.timing.doLater(conditional) is testing if the call should be done later. So it returns true if the param is false. arguments:

ParameterTypeDescription
conditionalanythingCan be a property that eventually gets set, or an expression, method... anything that can be evaluated.
contextObjectThe namespace where the call originated. Defaults to global and anonymous functions
intervalNumberPoll time to check conditional in Milliseconds
Example 1
setTimeout(function(){ if(dojox.timing.doLater(app.ready)){return;} console.log("Code is ready! anonymous.function SUCCESS") },700);
